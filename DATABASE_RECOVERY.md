# IndexedDB 數據庫恢復指南

## 問題描述

當 IndexedDB 被刪除（通常由以下原因導致）時，應用無法正常初始化：

- 用戶清空了瀏覽器快取或本地存儲
- 瀏覽器隱私模式清除了臨時數據
- 用戶手動刪除了數據庫
- 開發者工具中的 Application → IndexedDB 被清空

原始錯誤信息：

```
應用初始化失敗：The requested version (1) is less than the existing version (2)
```

## 解決方案

### 1. **自動重建機制** ✅

- 當 IndexedDB 被刪除後，應用會自動檢測到版本不匹配
- 系統會自動以「自動版本模式」重新打開數據庫
- IndexedDB 會自動觸發 `onupgradeneeded` 事件重建數據庫結構

### 2. **改進的初始化流程**

在 `database.js` 中的改進：

```javascript
// 檢測版本不匹配並自動恢復
if (errorMessage.includes("less than") && requestedVersion !== undefined) {
  console.log("檢測到版本不匹配，嘗試以自動版本模式重新打開...");
  openRequest(undefined); // 使用自動版本
}
```

### 3. **增強的錯誤提示和恢復**

在 `app.js` 中的改進：

- 詳細的初始化日誌，便於診斷問題
- 應用狀態指示器（在導航欄顯示：✅ 應用已就緒 / ❌ 初始化失敗）
- 一鍵重新初始化：點擊狀態指示器即可觸發重新初始化

### 4. **用戶友好的恢復流程**

```
發生錯誤 → 應用顯示初始化狀態 → 用戶點擊狀態指示器 → 自動重新初始化
```

## 技術細節

### 數據庫結構自動恢復

當 `onupgradeneeded` 事件被觸發時（數據庫版本升級或重新創建），系統會自動建立：

- ✓ `versions` ObjectStore - 存儲版本記錄和索引
- ✓ `tags` ObjectStore - 存儲版本標籤
- ✓ `comments` ObjectStore - 存儲批註信息
- ✓ `metadata` ObjectStore - 存儲元數據

### 版本鏈保護

- 版本鏈使用 `parentVersionId` 關聯，支持向後恢復
- 每個版本都有內容哈希驗證，確保數據完整性
- 支持新格式（完整內容）和舊格式（差異模式）的兼容

## 常見問題

### Q: 數據會被永久刪除嗎？

A: 是的。如果用戶清空了瀏覽器數據，IndexedDB 中的版本歷史會被完全刪除。應用會自動重建空的數據庫結構，但之前保存的版本無法恢復。**建議定期導出數據進行備份。**

### Q: 能否防止 IndexedDB 被刪除？

A: 不能。用戶有權清除浏览器數據。建議通過導出功能定期備份重要的版本歷史。

### Q: 應用為什麼顯示「⏳ 正在初始化...」？

A: 這表示初始化過程正在進行中。如果長時間停留在此狀態，請：

1. 查看瀏覽器 Console 的錯誤信息
2. 嘗試刷新頁面
3. 清除瀏覽器快取並重新加載

### Q: 如何手動清空數據庫進行測試？

A: 在瀏覽器開發者工具中：

1. 打開 Application 標籤
2. 在左側找到 IndexedDB
3. 右鍵點擊 `SQLVersionControl` 數據庫
4. 選擇「Delete Database」
5. 刷新頁面 - 應用會自動重建
