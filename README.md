# SQLScriptManage 使用手冊 📝

SQLScriptManage 是一個用瀏覽器就能使用的 SQL 腳本版本管理工具。你可以把不同 SQL 腳本分門別類保存起來，記錄每次修改，之後也能快速比較兩個版本差在哪裡。

它適合用在：

- 🧾 管理資料庫修改腳本
- 🔍 比較 SQL 前後版本差異
- 📦 匯出或備份自己的 SQL 版本紀錄
- 🧑‍💻 在沒有後端系統的情況下，簡單追蹤 SQL 修改歷程

## 🚀 開始使用

建議用本機伺服器開啟，不要直接雙擊 `index.html`，這樣瀏覽器儲存資料會比較穩定。

### 方法一：使用 Python

在專案資料夾執行：

```bash
python -m http.server 8000
```

然後用瀏覽器開啟：

```text
http://127.0.0.1:8000/
```

### 方法二：使用 npx

在專案資料夾執行：

```bash
npx serve .
```

依照畫面顯示的網址開啟即可。

## 🏠 主畫面介紹

開啟後會看到主要編輯畫面，大致分成幾個區域：

- 📁 **專案選單**：切換不同專案，讓不同工作內容分開管理。
- 📄 **SQL 腳本選單**：同一個專案底下可以管理多支 SQL。
- ✍️ **SQL 編輯器**：輸入或修改 SQL 內容。
- 🕘 **版本列表**：查看已保存的版本紀錄。
- 🔎 **版本詳情**：點選版本後，可以看到標籤、作者、描述與時間。
- 🧭 **上方工具列**：保存、格式化、下載 SQL、比較、導出、導入、備份與還原。

## 📁 管理專案

專案可以用來區分不同系統、客戶、資料庫或工作主題。

### 建立新專案

1. 點選「新建」。
2. 輸入專案名稱。
3. 建立完成後，系統會自動切換到新專案。

### 切換專案

在「專案」下拉選單中選擇要使用的專案。

### 刪除專案 ⚠️

點選「刪除」後依照提示選擇要刪除的專案。刪除專案會連同該專案內的 SQL 腳本與版本一起移除，操作前請先確認是否已備份。

## 📄 管理 SQL 腳本

每個專案底下可以保存多支 SQL 腳本，例如：

- `建立客戶資料表`
- `新增訂單欄位`
- `修正報表查詢`
- `初始化測試資料`

你可以在 SQL 腳本選單中切換目前正在編輯的腳本。切換後，版本列表也會跟著顯示該 SQL 腳本的版本紀錄。

## ✍️ 編輯 SQL

在中間的編輯器輸入 SQL。你可以像一般程式碼編輯器一樣編輯內容。

常用操作：

- 🧹 點選「格式化」整理 SQL 排版。
- ⬇️ 點選「下載 SQL」把目前編輯器內容下載成 `.sql` 檔案。
- 🌙 點選主題按鈕切換亮色或暗色介面。
- 🧭 底部狀態列會顯示游標位置、選取字數與儲存空間使用量。

## ⬇️ 下載 SQL

「下載 SQL」會把目前 SQL 編輯器中的內容下載成單一 `.sql` 檔案，檔名會優先使用目前 SQL 腳本名稱。

這個功能只下載當前編輯器內容，不會建立版本紀錄。若要保留修改歷史，請另外點選「保存版本」；若要備份版本資料，請使用「導出」或「完整備份」。

## 💾 保存版本

當 SQL 修改到一個段落後，可以保存成新版本。

1. 點選「保存版本」。
2. 輸入「版本標籤」，例如 `v1.0`、`新增索引`、`修正查詢條件`。
3. 輸入「作者」。
4. 視需要填寫「描述」，說明這次改了什麼。
5. 點選「保存版本」完成。

保存後，右側版本列表會出現新的紀錄。建議每次保存時都寫清楚標籤與描述，之後比較版本會更容易看懂。

## 🕘 查看與回到舊版本

在右側「版本列表」點選任一版本，就可以查看該版本內容與詳細資訊。

如果你想參考舊版本：

1. 點選版本列表中的版本。
2. 編輯器會載入該版本的 SQL 內容。
3. 若要保留目前修改，請再保存成新版本。

提醒：載入舊版本是把內容放回編輯器，不代表其他版本會被自動刪除。

## 🔍 比較版本差異

### 在主畫面比較

1. 點選「比較模式」。
2. 在左右兩側各選一個版本。
3. 畫面會顯示兩個版本的差異。

### 開啟獨立比較頁

也可以從「更多」選單開啟差異比對頁，選擇兩個版本後查看更完整的比較畫面。

差異顏色通常代表：

- 🟢 新增內容
- 🔴 刪除內容
- ⚪ 沒有變更的內容

## 📤 導出資料

「導出」適合只匯出某個專案的版本歷史。

1. 點選「導出」。
2. 選擇要導出的專案。
3. 選擇是否包含標籤與批註。
4. 點選「開始導出」。
5. 系統會下載一個 JSON 檔案。

這個檔案可以之後再導入回工具中。

## 📥 導入資料

「導入」可以把之前導出的 JSON 檔案匯入目前工具。

1. 點選「導入」。
2. 選擇 JSON 檔案。
3. 選擇要導入到哪個專案。
4. 如果遇到版本衝突，依照需求選擇處理方式。

衝突處理方式：

- ⏭️ **跳過**：保留目前資料，不匯入衝突版本。
- ♻️ **覆蓋**：用匯入檔案取代目前已有版本。
- ➕ **合併為新版本**：保留目前資料，將匯入內容建立成新的版本。

不確定時，建議選「合併為新版本」，比較不容易覆蓋既有資料。

## 💾 完整備份

「完整備份」會匯出全部資料，包含所有專案、SQL 腳本、版本、標籤與批註。

建議在以下情況先做完整備份：

- 🧹 清理瀏覽器資料前
- 💻 換電腦或換瀏覽器前
- 🔄 大量導入或還原資料前
- 🗓️ 定期保存重要工作紀錄

操作方式：

1. 點選「更多」。
2. 點選「完整備份」。
3. 確認備份內容。
4. 點選「開始備份」。
5. 保存下載的 JSON 檔案。

## ♻️ 完整還原

「完整還原」可以把完整備份檔案匯回工具中。

1. 點選「更多」。
2. 點選「完整還原」。
3. 選擇備份 JSON 檔案。
4. 選擇還原策略。
5. 點選「開始還原」。

還原時請特別注意：

- ✅ **合併還原**：保留目前資料，再加入備份內容。
- ⚠️ **清空後還原**：會先刪除目前所有資料，再匯入備份檔案。

如果不是非常確定，建議不要勾選「清空所有現有資料」。

## 🧹 刪除資料

工具中有幾種刪除操作：

- 🗑️ 刪除 SQL 腳本
- 🗑️ 刪除版本
- 🗑️ 刪除專案
- 🧨 刪除全部資料

刪除後通常無法直接復原。重要資料請先使用「完整備份」。

## 💡 使用建議

- 每次重要修改後都保存一個版本。
- 版本標籤要簡短清楚，例如 `新增欄位`、`修正 JOIN`、`上線前版本`。
- 描述可以寫修改原因，之後查找會更方便。
- 定期使用「完整備份」下載 JSON 檔案。
- 匯入陌生資料前，先備份目前資料。
- 不同系統或客戶建議分成不同專案。

## ⚠️ 重要提醒

- 資料保存在目前瀏覽器本機，不會自動上傳到雲端。
- 清除瀏覽器網站資料後，工具內的專案與版本可能會消失。
- 換瀏覽器、換網址或換電腦時，原本資料不會自動出現。
- 需要搬移資料時，請使用「完整備份」與「完整還原」。
- 若網路無法連線，部分外部編輯器功能可能無法正常載入。

## ❓ 常見問題

### Q：我的資料會傳到伺服器嗎？

不會。資料主要保存在你的瀏覽器本機。

### Q：可以多人共用同一份資料嗎？

這個工具主要是單機使用。若要分享資料，可以使用「導出」或「完整備份」產生 JSON 檔案，再交給其他人導入。

### Q：我清掉瀏覽器資料後版本不見了，怎麼辦？

如果之前有做完整備份，可以用「完整還原」匯回。如果沒有備份，通常無法復原。

### Q：導入和完整還原有什麼不同？

「導入」比較適合匯入單一專案的版本歷史；「完整還原」適合恢復整個工具的所有資料。

### Q：比較頁打不開怎麼辦？

請先確認已經保存至少兩個版本，並且是透過本機伺服器開啟工具，而不是直接用 `file://` 開啟。
